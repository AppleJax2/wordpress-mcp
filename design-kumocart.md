## User Flow

1. **Discovery**
   - User lands on the KumoCart homepage
   - Reviews features and pricing
   - Reads documentation on integration capabilities

2. **Signup & Onboarding**
   - Creates an account
   - Selects a subscription plan
   - Completes payment (if applicable)
   - Guided to API key generation

3. **Integration**
   - Generates first API key
   - Follows integration guide to connect with Claude Desktop or Cursor IDE
   - Tests connection with simple WordPress automation task

4. **Usage & Management**
   - Monitors usage stats in dashboard
   - Manages API keys (create, revoke)
   - Upgrades plan as needed
   - Accesses support for questions

## Routes/Views

- **Public Views**
  - `/` - Homepage with product info
  - `/features` - Detailed product features
  - `/pricing` - Subscription plans and pricing
  - `/docs` - Public documentation
  - `/login` - User login
  - `/signup` - New account registration

- **Authenticated Views**
  - `/dashboard` - User dashboard overview
  - `/api-keys` - API key management
  - `/usage` - Usage statistics and analytics
  - `/billing` - Subscription and payment management
  - `/settings` - Account settings
  - `/support` - Support ticket system

## API Key Management System

- **Features**
  - Multiple key generation per account
  - Key masking for security (only showing last 4 characters)
  - Creation timestamp tracking
  - One-click copy functionality
  - Key revocation
  - Usage tracking per key
  - Rate limiting based on subscription tier

- **Security Measures**
  - Keys stored with encryption at rest
  - Auth headers required for all API calls
  - IP-based request validation (optional)
  - Automatic suspicious activity detection
  - Admin alert system for unusual usage patterns
